Java Application to Run and Store DBMS Queries

Technologies Used: Java, SQL (Oracle), JDBC

Summary: Developed a Java Application which executes SQL queries against a relational database and places the results in special data structures.

Background and Premise:

This project uses a fictional Facebook-like database with tables that stores different types of user data, including location, friends, educational history, tagged photos, etc. Because the contents of the database were organized in differing data forms, I was to create efficient Java functions that both extracted and stored the results into their appropriate structures. The task of querying this database through Java required relevant knowledge of the schema of the database and a familiarity of the methods of Java Database Connectivity. Nine queries were performed on the database, each ranging in complexity. Statements and ResultSet operations were among the most widely employed throughout the project.

Results:

    Completion of the following complex queries:

  • Returned the top "num" pairs of users with the most mutual friends who are not friends themselves.
  • Returned pairs of "unrealized" friendships in which the following conditions had to be met:
    • The two users are the same gender
    • The two users are tagged in at least one common photo
    • The two users are not friends

    *Results were sorted in descending order by number of photos tagged together.

  • Returned the top "num" photos with the most tagged users sorted in descending order by the number of tagged users.

Implementation of Grace-Hash Join in OOP Environment

Image Source: UNSW

Technologies Used: C++, Relational DBMS

Summary: Emulated the Grace Hash Join Algorithm for relational databases in C++.

Background and Premise:

Grace Hash Join is a join algorithm that is used to join two relations in a database. It is a disk-based join algorithm that is used when the relations are too large to fit in memory. The algorithm has two main phases: partition and probe. In the partition phase, the algorithm partitions the relations into buckets based on a hash function. In the probe phase, the algorithm probes the buckets to find matching tuples and re-hashes if necessary. In this project, I implemented the partition and probe functions of the Grace Hash Join algorithm in C++. I was given starter files that simulated the data flow of records in disk and memory and performed a join operation between two relations.

Results:

  • Implemented the partition and probe functions of the Grace Hash Join algorithm in C++
  • Successfully joined two relations using the Grace Hash Join algorithm
  • Tested the implementation with different input sizes and configurations
  • Optimized the implementation for performance and efficiency
  • Proved the ability to implement a join algorithm in an OOP environment based on its written description and purpose
